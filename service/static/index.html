ARP 引流条目（逐条配置）

每条记录：冒充哪个网关 IPv4、经 Linux 200 的哪个接口发二层帧（真实 MAC 取自该口）；守护进程对该条做周期性 GARP并在收到 ARP 询问时按策略回复。保存后 OP 会立即尝试在该接口上添加该 IPv4 的 /32（与守护进程逻辑一致，便于 BGP update-source）；总开关关闭或删除条目时会撤掉不再需要的 /32。需本机具备 CAP_NET_ADMIN（通常 root 跑 OP）。可用环境变量 MTR_OP_ARP_ASSIGN_HOST_IP=0 关闭 OP 侧下发。总览可一键开关 ARP 引流总控；单条仍以「启用」为准。卫星 VRF 自动建：在 200 上设 MTR_AUTO_SATELLITE_VRF=cidr（或 all / note）后，保存匹配条目的 ARP 即会尝试创建 vbgp*（见 service/README.md）；亦可点击上方「同步卫星VRF」按钮。

VPN 出口（OpenVPN / GRE / L2TP → vrf2103）

数据存 SQLite；下发调用 POST /api/vpn/apply（Linux 上执行 ip / OpenVPN）。非 Linux 或设置 MTR_OP_VPN_APPLY=0 时仅更新库状态。说明：本功能为本机连外部 VPN 作出口，非用户拨入。
config 示例 — GRE：{"gre":{"remote":"1.2.3.4","local":"0.0.0.0","mtu":1476}}； OpenVPN：{"remote":"vpn.example.com:1194","proto":"udp","ca":"/path/ca.crt",...}； L2TP：{"l2tp":{"server":"1.2.3.4","username":"u","password":"***","ipsec_psk":"***"}}（密码建议用 password_file 绝对路径；下发后在 $MTR_OP_DATA/vpn/l2tp-<id>/ 取合并片段）。
