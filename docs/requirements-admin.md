# vRouter 管理平台需求文档（汇总版）

更新时间：2026-05-08  
范围：管理后台（Web）+ 管理 API（对接后端服务）

---

## 1. 背景与目标

`vRouter` 部署在 Ubuntu 服务器上（Docker 运行），配套 FRR（BGP）、Redis（规则库/缓存）、MySQL（配置/日志/审计）。管理平台用于统一完成 **配置、监控、查询、审计、运维** 等工作，并对外提供“路由表查询”能力。

---

## 2. 总体能力清单（按模块）

### 2.1 系统总览与运行状态

- **组件健康**：vRouter、FRR/BGP、Redis、MySQL 的运行状态与健康度
- **关键指标**：CPU、内存、链路/转发概览、MTR 命中与丢包概览、VPN 在线/异常统计
- **告警中心**：展示最近告警（BGP 断开、VPN 异常、策略异常等），支持确认/清空（一期可做清空/确认；二期可做告警规则与通知）

### 2.2 BGP 管理（区分上游/下游）

- **邻居管理**：新增/编辑/删除 BGP 邻居
  - 邻居 IP、Remote AS、Router ID（可选）、Keepalive/Hold（可选）
- **Peer 角色区分**
  - **上游运营商 Peer**：用于从上级设备学习路由（例如 RR/运营商）
  - **下游客户 Peer**：用于客户侧互联（例如 C3/C4）
- **状态监控**：邻居状态（Established/Down 等）、学习路由条数、更新时间、状态变更记录
- **邻居详情**：收/发前缀统计、基础属性、诊断信息（一期可仅展示；二期补充操作）
- **导出能力**：导出 BGP 邻居与状态（用于排障）

### 2.3 路由表查询（实时 + 缓存回退）

- **查询条件**：按前缀/关键字查询
- **数据来源选择**
  - 自动（优先实时 BGP）
  - 强制实时 B
  - 强制缓存 BGP
- **返回标记与状态透出**
  - `source=live_bgp` / `source=cached_bgp`
  - BGP 当前状态：up/down
  - 缓存时间：`cache_time`
- **路由属性展示**：prefix、next-hop、来源邻居、AS_PATH、LOCAL_PREF、MED 等
- **详情/导出**：路由详情、导出路由表

### 2.4 缓存路由管理（用于“查询回退”）

- **缓存快照展示**：缓存版本号、缓存时间、缓存条目列表
- **缓存触发**：定时缓存（建议）+ 手动“立即缓存”（一期可模拟）
- **缓存存储**：Redis（热）+ MySQL（冷/审计/历史）
- **回退逻辑**：
  - 当与上游设备 BGP 断开时，客户查询路由表 → 返回最近一次缓存快照
  - 返回需标记 `source=cached_bgp`，并带 `cache_time`
- **说明**：缓存用于“查询回退”，不等价于继续按缓存路由转发（如需转发需单独需求）

### 2.5 MTR/ICMP 劫持策略管理

- **总开关**：开启/关闭劫持能力
- **策略管理**：新增/编辑/启停策略
- **匹配条件**：运营商、目标网段（CIDR）等
- **应答行为**：伪造返回 IP、TTL 策略、延迟模拟策略
- **统计与日志**：命中次数、延迟统计、命中明细

### 2.6 IP 运营商规则库管理（Redis）

- **导入/导出**：导入用户提供的 IP-运营商库；导出备份
- **在线查询**：输入 IP 查询运营商归属（用于验证策略）
- **规则管理**：查看 IP 段、运营商、版本、更新时间；支持启停/删除（需审计）
- **版本化（建议）**：每次导入生成版本号，可回滚，记录导入审计

### 2.7 VPN 出口管理（外部访问能力）

说明：这里 VPN 是 **本路由器连接外部 VPN 服务作为出口访问外部服务**，不是“用户拨入”。

- **支持类型**：OpenVPN、GRE、L2TP
- **隧道管理**：新增/编辑/启停、优先级、状态（在线/断开/异常）、延迟与流量统计
- **连通性测试**：对外部 VPN 端点或外部服务进行连通性检测
- **故障策略**：主备切换、回退普通出口或拒绝转发（防止流量泄露）

### 2.8 VPN 路由策略管理（策略路由）

- **策略维度**：目的网段、来源客户（或接口/源地址）
- **选择出口**：绑定到具体 VPN（OpenVPN/GRE/L2TP）
- **失败策略**：切换备用 VPN / 回退普通出口 / 拒绝转发
- **策略启停/优先级**：启停、优先级（可扩展命中统计）

### 2.9 日志审计与可观测性

- **日志分类**
  - BGP 状态变化日志
  - 路由缓存生成/回退日志
  - MTR 劫持命中日志
  - VPN 状态与切换日志
  - 登录/权限与操作审计日志
- **检索能力**：按类型筛选、关键字搜索
- **详情/导出**：日志详情查看、导出

### 2.10 系统设置与权限（RBAC）

- **基础参数**：Router ID、Local AS、缓存周期、日志保留周期等
- **用户与角色**：管理员/运维/审计（只读/可写权限）
- **环境信息**：接口信息、组件版本、运行环境概览（用于运维）

## 4. 关键数据字段（最小集合）

### 4.1 BGP 邻居（Peer）

- `peer_ip`：邻居 IP/接口标识
- `remote_as`：Remote AS
- `role`：`upstream` / `downstream`
- `owner`：上游运营商名称 / 客户名称
- `state`：Established/Down/Idle/Active 等
- `routes_received`：学习路由条数
- `updated_at`：更新时间

### 4.2 路由记录（Route）

- `prefix`
- `next_hop`
- `neighbor`（来源邻居）
- `as_path`
- `local_pref`
- `med`
- `source`：`live_bgp` / `cached_bgp`
- `cache_time`（当 source 为 cached 时）

### 4.3 MTR/ICMP 策略（Policy）

- `id`
- `isp_match`
- `dst_cidr`
- `reply_ip`
- `ttl_policy`
- `delay_policy`
- `enabled`

### 4.4 IP 运营商规则（IPDB）

- `cidr`
- `isp`
- `version`
- `updated_at`

### 4.5 VPN 隧道（VPN Link）

- `name`
- `type`：OpenVPN / GRE / L2TP
- `endpoint`
- `status`：up/down/warn
- `priority`
- `rtt_ms`
- `traffic`

### 4.6 VPN 路由策略（VPN Policy）

- `dst_cidr`
- `src_scope`（客户/接口/源网段）
- `vpn_name`
- `fail_action`：切换/回退/拒绝
- `enabled`

### 4.7 审计日志（Audit Log）

- `ts`
- `type`
- `object`
- `message`
- `operator`（可选）

## 5. 权限与审计（建议）

- **角色**
  - 超级管理员：全功能
  - 运维：配置 + 状态查看
  - 审计：只读 + 导出
- **审计要求**
  - 配置变更（BGP/VPN/策略/规则库）必须记录操作人、时间、变更内容
  - 登录/登出、失败登录、权限变更需记录

---

## 6. 非功能需求（建议）

- **安全**：HTTPS、强密码、最小权限、关键操作二次确认（可选）
- **可用性**：前端容灾（静态资源）、后端健康检查与重启策略
- **性能**：路由表查询分页/导出限流；规则库查询高并发（Redis）
- **可运维性**：关键指标与日志可导出；支持问题定位（BGP/VPN/MTR 命中链路）
