# MTR-OP 部署说明（上传代码 + 重启服务）

本文档面向**日常发版**：把仓库代码同步到目标机，重启相关进程即可。  
**不**在每次发版时执行 `apt-get`、重装 Go、清空远端目录。

架构见 [BGP_ARCHITECTURE.md](./BGP_ARCHITECTURE.md)，表/API 见 [BGP_DATA_AND_API.md](./BGP_DATA_AND_API.md)，部署验收见 [BGP_RXTX_DEPLOYMENT.md](./BGP_RXTX_DEPLOYMENT.md)。**功能操作步骤与预期效果**见 [OP_OPERATION_MANUAL.md](./OP_OPERATION_MANUAL.md)。

**新机 / 迁回现网全量（含清库与条件安装）**：见 [`109/MIGRATION.md`](../109/MIGRATION.md)（先审 [`109/env.example`](../109/env.example) 再执行 `109/deploy_fresh.py`）。

## 2. 需要上传哪些文件

从**仓库根目录**出发，同步到远端 **`/root/mtr_op/`**（目录内容与本地 `service/` 一致）。

| 本地路径 | 远端路径 |
|----------|----------|
| `service/` 下全部（递归） | `/root/mtr_op/` |
| `scripts/te_rewrite_nfqueue.py` | `/root/mtr_op/te_rewrite_nfqueue.py` |

**不要上传**（保留远端已有）：

- `venv/`、`.venv/`、`__pycache__/`
- `*.db`（如 `data.db`，避免覆盖现网数据）
- 已编译二进制 `service/bgp_agent/bgp_agent`（改 Go 代码时在**本机** `python tools/bgp_agent_build.py`）

## 3. 上传代码

任选一种方式，在**开发机**执行。

### 3.1 rsync（推荐，Linux / WSL / macOS）

```bash
export VR_IP=101.89.68.109
export SSH_USER=root

rsync -avz --delete \
  --exclude 'venv' --exclude '.venv' --exclude '__pycache__' \
  --exclude '.git' --exclude '*.db' --exclude 'bgp_agent/bgp_agent' \
  service/ ${SSH_USER}@${VR_IP}:/root/mtr_op/

rsync -avz scripts/te_rewrite_nfqueue.py \
  ${SSH_USER}@${VR_IP}:/root/mtr_op/te_rewrite_nfqueue.py
```

### 3.2 scp（Windows / 无 rsync 时）

```powershell
$VR = "101.89.68.109"
scp -r service\* root@${VR}:/root/mtr_op/
scp scripts\te_rewrite_nfqueue.py root@${VR}:/root/mtr_op/
```

### 3.3 一键轻量部署（Python + Paramiko，推荐日常）

```powershell
# 仓库根目录
$env:MTR_OP_HOST = "101.89.68.109"
$env:MTR_OP_SSH_PASSWORD = 't$7kaptj+A)J89NB'
python tools/deploy_light.py

# 仅更新 OP / TE / 静态页（不上传、不重启 bgp-agent）
python tools/deploy_light.py --op-only

# 仅本机编译并更新 bgp-agent（不重启 uvicorn）
python tools/deploy_light.py --agent-only
```

`deploy_light.py` 会：上传 `service/` 与 `te_rewrite_nfqueue.py`、清理遗留 **`mtr_spoof_nfqueue`**、应用 **`nft_mtr_te.nft`**、按环境重启 uvicorn（及默认同步 bgp-agent）。**只改 Python/规则** 用 **`--op-only`**；**只改 Agent** 用 **`--agent-only`**。

### 3.4 新机全量安装

```powershell
$env:MTR_OP_HOST = "101.89.68.109"
$env:MTR_OP_SSH_PASSWORD = 't$7kaptj+A)J89NB'
$env:RR_ADDR = "139.159.43.249"
$env:RR_AS = "63199"
$env:LOCAL_AS = "63199"
$env:ROUTER_ID = "139.159.43.207"
$env:MTR_SATELLITE_PEER_IP = "139.159.43.208"
python service/scripts/deploy_bgp_rxtx.py
```

> 日常迭代请用 **`deploy_light.py`** 或 rsync + §4，**勿**在未备份时全量 `rm -rf /root/mtr_op`。  
> 若必须跑全量脚本且要保留库，可加：`$env:MTR_OP_PRESERVE_DIR = "1"` 与 `$env:MTR_OP_SKIP_INSTALL = "1"`。

## 4. 重启服务

上传完成后 SSH 登录目标机。

### 4.1 仅改了 Python / 前端静态资源

```bash
ssh root@101.89.68.109
cd /root/mtr_op

export GOBGP_AGENT_URL=http://127.0.0.1:9179
export MTR_OP_DB=/root/mtr_op/data.db
export MTR_OP_NFT=/root/mtr_op/nft_mtr_te.nft
export MTR_OP_DATA=/root/mtr_op/data
export MTR_TE_REWRITE_SCRIPT=/root/mtr_op/te_rewrite_nfqueue.py

pkill -f 'uvicorn app.main' 2>/dev/null || true
pkill -f mtr_spoof_nfqueue 2>/dev/null || true
sleep 1

nft delete table inet mtr_spoof 2>/dev/null || true
nft delete table inet mtr_te 2>/dev/null || true
[ -f nft_mtr_te.nft ] && nft -f nft_mtr_te.nft || true

: > /tmp/mtr_op.log
nohup ./venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8808 >> /tmp/mtr_op.log 2>&1 &
# TE 改写守护由 uvicorn 启动时 te_rewrite_sync 按 hijack_enabled 拉起
```

`bgp-agent` **无需**重启。

### 4.2 GoBGP Agent（`bgp-agent.service`）

**日常发版已自动化**：`deploy_light.py` 会写入 systemd（`RR_ADDR` / `LOCAL_AS` / `ROUTER_ID` 等）、`systemctl restart bgp-agent`，并 `curl` 验收 `9179/health` 与 `9179/api/status`。

仅同步 Agent、不上传 OP 代码：

```powershell
$env:MTR_OP_HOST = "101.89.68.109"
$env:MTR_OP_SSH_PASSWORD = '<密码>'
$env:RR_ADDR = "139.159.43.249"
$env:RR_AS = "63199"
$env:LOCAL_AS = "63199"
$env:ROUTER_ID = "139.159.43.207"
python tools/sync_bgp_agent.py
```

**默认**：`109/env` 已设 `MTR_DEPLOY_BGP_AGENT_PREBUILT=1`、`MTR_DEPLOY_BUILD_BGP_AGENT=0`——本机（WSL/Docker）编译后上传，**不在 VR 上 go build**。

```powershell
python tools/deploy_light.py
# 改了 service/bgp_agent Go 代码时：
python tools/bgp_agent_build.py
python tools/deploy_light.py
```

仅同步 unit、不编译不上传二进制：`python tools/sync_bgp_agent.py`（默认 `MTR_DEPLOY_BUILD_BGP_AGENT=0`）。

恢复 VR 编译（不推荐）：`MTR_DEPLOY_BUILD_BGP_AGENT=1` 且 `MTR_DEPLOY_BGP_AGENT_PREBUILT=0`。

WSL 内安装 RocksDB 开发库（首次）：

```bash
sudo apt install -y librocksdb-dev libsnappy-dev zlib1g-dev libbz2-dev liblz4-dev libzstd-dev
```

环境变量：`MTR_DEPLOY_BGP_AGENT_PREBUILT=1` 上传预编译包并跳过远端 build；`MTR_DEPLOY_BGP_AGENT_LOCAL_BUILD=0` 表示不自动编译、仅上传已有二进制。

手工等价命令（排错用）：

```bash
systemctl restart bgp-agent
curl -s http://127.0.0.1:9179/health
curl -s http://127.0.0.1:9179/api/status
```

### 7.1 MTR / TE 改写（109 常见）

| 检查 | 命令 / 说明 |
|------|-------------|
| 总开关 | `curl -s http://127.0.0.1:8808/api/global` → `hijack_enabled` |
| 守护进程 | `pgrep -af te_rewrite_nfqueue` |
| 队列监听 | `cat /proc/net/netfilter/nfnetlink_queue`（空则无监听，TE 可能被丢） |
| iptables | `iptables -t mangle -S FORWARD`（应为 **`icmp-type 11`** → NFQUEUE） |
| 遗留路径 B | 不应存在 `mtr_spoof_nfqueue`；`nft list table inet mtr_spoof` 应为空 |

**mtr 全 timeout**：多为 **NFQUEUE 无监听** 或守护启动失败；**关总开关后仍 timeout** 多为旧版 iptables 删不干净。处理步骤与根因表见 **[MTR_TE_REWRITE.md](./MTR_TE_REWRITE.md)**。

**逐跳规则不生效**：确认 `hijack_enabled=true`、`match_cidr` 与真实 TE **外层源** 一致；109 下游路径见 [MTR_DOWNSTREAM_TRANSIT_109.md](./MTR_DOWNSTREAM_TRANSIT_109.md)。

§4.1 中 `pkill mtr_spoof_nfqueue`、`nft delete inet mtr_spoof` 仅用于**清理历史遗留**；现网逻辑以 `te_rewrite_nfqueue` + `inet mtr_te` 为准。
