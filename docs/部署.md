# MTR-OP 部署说明（上传代码 + 重启服务）

本文档面向**日常发版**：把仓库代码同步到目标机，重启相关进程即可。  
**不**在每次发版时执行 `apt-get`、重装 Go、清空远端目录。

## 2. 需要上传哪些文件

从**仓库根目录**出发，同步到远端 **`/root/mtr_op/`**（目录内容与本地 `service/` 一致）。

| 本地路径 | 远端路径 |
|----------|----------|
| `service/` 下全部（递归） | `/root/mtr_op/` |
| `scripts/mtr_spoof_nfqueue.py` | `/root/mtr_op/mtr_spoof_nfqueue.py` |

**不要上传**（保留远端已有）：

- `venv/`、`.venv/`、`__pycache__/`
- `*.db`（如 `data.db`，避免覆盖现网数据）
- 已编译二进制 `bgp_agent/bgp_agent`（改 Go 代码时在**远端**重新 `go build` 即可）

## 3. 上传代码

任选一种方式，在**开发机**执行。

### 3.1 rsync（推荐，Linux / WSL / macOS）

```bash
export VR_IP=101.89.68.109
export SSH_USER=root

rsync -avz --delete \
  --exclude 'venv' --exclude '.venv' --exclude '__pycache__' \
  --exclude '.git' --exclude '*.db' --exclude 'bgp_agent/bgp_agent' \
  service/ ${SSH_USER}@${VR_IP}:/root/mtr_op/

rsync -avz scripts/mtr_spoof_nfqueue.py \
  ${SSH_USER}@${VR_IP}:/root/mtr_op/mtr_spoof_nfqueue.py
```

### 3.2 scp（Windows / 无 rsync 时）

```powershell
$VR = "101.89.68.109"
scp -r service\* root@${VR}:/root/mtr_op/
scp scripts\mtr_spoof_nfqueue.py root@${VR}:/root/mtr_op/
```

### 3.3 一键轻量部署（Python + Paramiko，推荐日常）

```powershell
# 仓库根目录
$env:MTR_OP_HOST = "101.89.68.109"
$env:MTR_OP_SSH_PASSWORD = 't$7kaptj+A)J89NB'
python tools/deploy_light.py
```

### 3.4 新机全量安装

```powershell
$env:MTR_OP_HOST = "101.89.68.109"
$env:MTR_OP_SSH_PASSWORD = 't$7kaptj+A)J89NB'
$env:RR_ADDR = "139.159.43.249"
$env:RR_AS = "63199"
$env:LOCAL_AS = "63199"
$env:ROUTER_ID = "139.159.43.207"
$env:MTR_SATELLITE_PEER_IP = "139.159.43.208"
python service/scripts/deploy_bgp_rxtx.py
```

> 日常迭代请用 **`deploy_light.py`** 或 rsync + §4，**勿**在未备份时全量 `rm -rf /root/mtr_op`。  
> 若必须跑全量脚本且要保留库，可加：`$env:MTR_OP_PRESERVE_DIR = "1"` 与 `$env:MTR_OP_SKIP_INSTALL = "1"`。

## 4. 重启服务

上传完成后 SSH 登录目标机。

### 4.1 仅改了 Python / 前端静态资源

```bash
ssh root@101.89.68.109
cd /root/mtr_op

export GOBGP_AGENT_URL=http://127.0.0.1:9179
export MTR_OP_DB=/root/mtr_op/data.db
export MTR_OP_NFT=/root/mtr_op/nft_mtr_spoof.nft
export MTR_OP_DATA=/root/mtr_op/data

pkill -f 'uvicorn app.main' 2>/dev/null || true
pkill -f mtr_spoof_nfqueue 2>/dev/null || true
sleep 1

nft delete table inet mtr_spoof 2>/dev/null || true
[ -f nft_mtr_spoof.nft ] && nft -f nft_mtr_spoof.nft || true

: > /tmp/mtr_op.log
nohup ./venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8808 >> /tmp/mtr_op.log 2>&1 &

: > /tmp/mtr_spoof_nfqueue.log
nohup ./venv/bin/python mtr_spoof_nfqueue.py --op-db /root/mtr_op/data.db --verbose \
  >> /tmp/mtr_spoof_nfqueue.log 2>&1 &
```

`bgp-agent` **无需**重启。

### 4.2 GoBGP Agent（`bgp-agent.service`）

**日常发版已自动化**：`deploy_light.py` 会写入 systemd（`RR_ADDR` / `LOCAL_AS` / `ROUTER_ID` 等）、`systemctl restart bgp-agent`，并 `curl` 验收 `9179/health` 与 `9179/api/status`。

仅同步 Agent、不上传 OP 代码：

```powershell
$env:MTR_OP_HOST = "101.89.68.109"
$env:MTR_OP_SSH_PASSWORD = '<密码>'
$env:RR_ADDR = "139.159.43.249"
$env:RR_AS = "63199"
$env:LOCAL_AS = "63199"
$env:ROUTER_ID = "139.159.43.207"
python tools/sync_bgp_agent.py
```

跳过远程 `go build`（只改 unit 参数时）：`$env:MTR_DEPLOY_BUILD_BGP_AGENT = "0"`。

手工等价命令（排错用）：

```bash
systemctl restart bgp-agent
curl -s http://127.0.0.1:9179/health
curl -s http://127.0.0.1:9179/api/status
```

## 6. 环境变量

### 6.1 Python OP（§4 重启脚本 / systemd）

| 变量 | 说明 | 现网示例 |
|------|------|----------|
| `GOBGP_AGENT_URL` | Python 调 Go Agent | `http://127.0.0.1:9179` |

| `MTR_OP_NFT` | nft 规则文件 | `/root/mtr_op/nft_mtr_spoof.nft` |
| `MTR_OP_DATA` | 数据目录 | `/root/mtr_op/data` |
| `MTR_SATELLITE_PEER_IP` | 卫星/下游 BGP 对端 IP | `139.159.43.208` |
| `MTR_BGP_ROLE_MAP` | 邻居角色 | `139.159.43.249:rr,139.159.43.208:downstream` |
| `MTR_BGP_DB_PRESETS` | 入库预设 | `default:139.159.43.249:rr,default:139.159.43.208:downstream` |

### 6.2 BGP Agent（`bgp-agent.service` / 全量部署）

| 变量 | 现网值 |
